# Plan 08-01: Patrol Path Waypoint Editor

## Goal
When a creature spawn with MovementType=PATROL is selected in the spawn editor, the user can:
1. Click in the 3D viewport to add a new waypoint at that position
2. Drag an existing waypoint gizmo to reposition it
3. See a looping line path connecting all waypoints in world space
4. Edit each waypoint's wait time in the details panel
5. Delete individual waypoints

### `F:\mmo\src\mmo_edit\editors\world_editor\edit_modes\spawn_edit_mode.cpp`

#### In constructor / `OnDeactivate()`:
- Destroy any `m_waypointPathObj` and `m_waypointSpheres` render objects

#### `OnMouseDown(float x, float y)`:
- Only act if `m_selectedUnitSpawn != nullptr` and its `movement_type()` == 1 (PATROL)
- Raycast (x, y) to world position using `m_worldEditor.GetRaycastHit(x, y, hitPos)` (same pattern area trigger mode uses)
- Check `HitTestWaypoint(hitPos)` — if within ~1.5m of existing waypoint, begin drag (`m_draggingWaypointIndex`)
- Else: add new waypoint at hitPos via `m_selectedUnitSpawn->mutable_waypoints()->Add()`, set positionx/y/z, call `RebuildWaypointVisualization()`

#### `OnMouseMoved(float x, float y)`:
- If `m_draggingWaypointIndex >= 0`: raycast to world, update waypoint proto position, call `RebuildWaypointVisualization()`

#### `OnMouseUp(float x, float y)`:
- Clear `m_draggingWaypointIndex = -1`

#### `DrawDetails()` — in unit spawn details section:
- After movement type combo, if movement_type == PATROL:
  - `ImGui::SeparatorText("Patrol Waypoints");`
  - "Click in the viewport to add waypoints" helper text
  - For each waypoint i: show `[i+1]` label, `InputFloat("Wait(ms)", &wp.waittime)`, `SmallButton("X")` to delete
  - "Clear All" button
  - On any change: call `RebuildWaypointVisualization()`

#### `RebuildWaypointVisualization()`:
- Destroy existing `m_waypointPathObj` and `m_waypointSpheres`
- If no spawn selected or no waypoints: return
- Create `m_waypointPathObj` via `m_worldEditor.CreateManualRenderObject("WaypointPath")`
- Add line list operation with wireframe material
- Add lines: waypoint[0]→[1]→...→[n-1]→[0] (loop)
- For each waypoint: create a small sphere render object (or a cross-hair made of 3 short lines) marking the waypoint position

### `F:\mmo\src\mmo_edit\editors\world_editor\world_editor_instance.cpp`
- When a unit spawn is selected / deselected, notify `SpawnEditMode` so it can update `m_selectedUnitSpawn` and rebuild visualization
- Ensure `m_worldEditor.GetRaycastHit(x, y, outPos)` exists or add it (check if area_trigger_edit_mode already calls a similar method; if so, use the same)

## Waypoint Edit Activation Logic
- `m_waypointEditActive` is true whenever:
  - A unit spawn is selected (`m_selectedUnitSpawn != nullptr`)
  - AND its `movement_type()` == MovementType::Patrol (value 1)
- When a different spawn is selected or deselected, clear visualization and `m_waypointEditActive`

## Data Flow
1. User clicks viewport → `OnMouseDown` → adds waypoint to `m_selectedUnitSpawn->mutable_waypoints()`
2. `RebuildWaypointVisualization()` re-renders lines/spheres
3. User saves map → `WorldEditorInstance::Save()` serializes `UnitSpawnEntry` including waypoints to proto file

## UAT
1. Select a creature spawn, change movement type to PATROL → waypoint section appears in details panel
2. Click 3 times in viewport → 3 waypoints added, line loop drawn connecting them
3. Drag an existing waypoint sphere → waypoint position updates, line redraws in real time
4. Set wait time on waypoint 2 → value persists in proto after save/reload
5. Click "X" on waypoint 1 → removed, loop re-draws without it
6. Switch to Stationary movement type → waypoint section disappears, viz removed
7. Save map → reload → waypoints still present with correct positions and wait times
