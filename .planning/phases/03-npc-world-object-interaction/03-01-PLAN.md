**Problem:** `TrainerFrame_OnTrainerUpdate(self)` is an empty function body. When the server sends a TRAINER_UPDATE event (e.g., after BuyTrainerSpell succeeds or fails), the trainer list does not refresh and the buy button state does not update. The player sees a stale list.

**Fix — fill in the empty handler:**
```lua
function TrainerFrame_OnTrainerUpdate(self)
    TrainerList_Update(TrainerFrame);
end
```
Note: call `TrainerList_Update(TrainerFrame)` explicitly with `TrainerFrame` as argument, NOT `TrainerList_Update(self)`. The reason: when TRAINER_UPDATE fires, `self` is the TrainerFrame — which is correct in that context. However, `TrainerList_Update` is also registered directly as an event handler for MONEY_CHANGED, SPELL_LEARNED, PLAYER_LEVEL_CHANGED (lines 117-119), where `self` is also TrainerFrame. Using `TrainerFrame` explicitly is the safe, unambiguous choice that avoids nil errors if TrainerFrame_OnTrainerUpdate is ever called directly.
