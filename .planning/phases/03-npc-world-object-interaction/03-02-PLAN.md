<objective>
Implement the missing UseObject opcode: add it to the shared protocol enum, implement the client-side packet sender in RealmConnector, and replace the fragile spell-cast world object interaction in PlayerController with the new direct opcode.

Purpose: WOBJ-01 requires world objects to respond to player interaction. The server handler (Plan 03-03) depends on the opcode existing in the protocol enum. The current client code at player_controller.cpp:808-818 calls `GetOpenSpell()` and casts a spell to use world objects — this fails silently if the player has no "open" spell, breaking interaction for all world objects without a bound spell. The direct UseObject opcode removes this fragile dependency.

Output: `game::client_realm_packet::UseObject` enum value, `RealmConnector::UseObject(uint64)` method, and updated click handler in PlayerController.
</objective>
