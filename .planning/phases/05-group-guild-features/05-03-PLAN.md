**1a. party_info.cpp — Fix 2-person group bug (Pitfall 1)**

Read `party_info.cpp` around line 248. Find the condition `if (members.size() > 1)` that guards the loot method read block. Change it to `if (memberCount > 0)` where `memberCount` is the member count read from the packet. The server sends `memberCount = N-1` (excluding self), so for a 2-person group `memberCount = 1` and `1 > 1 == false` — the bug. Changing to `memberCount > 0` makes it `1 > 0 == true`.

```cpp
// OLD (buggy):
if (members.size() > 1)

// NEW (fixed):
if (memberCount > 0)
```
