<objective>
Complete the client-side C++ chat routing: guard against the name-cache crash on system messages, add the RAID type string mapping, and fire the local whisper confirmation event.

Purpose: Two client-side C++ gaps remain after Plan 01 and Plan 02:

1. `world_state.cpp OnChatMessage` calls `m_cache.GetNameCache().Get(characterGuid, ...)` for all non-NPC types. When the server sends a system message (`ChatType::System`) with `characterGuid = 0`, the name cache either fires with an empty name (displaying malformed text) or hangs waiting for a server name query that never resolves. An early-return guard is required before the name cache call.

2. `game_script.cpp` `s_typeStrings[]` has no `{"RAID", ChatType::Raid}` entry, so `SendChatMessage(msg, "RAID")` from Lua prints "Unknown chat type 'RAID'!" and silently discards the message — raid chat is impossible even after Plan 01 adds the server handler.

3. `game_script.cpp SendChatMessage()` does not fire `CHAT_MSG_WHISPER_INFORM` after routing a whisper. Without this, the sender never sees their own whisper in the chat frame (the Plan 02 handler registers for the event but nothing fires it).

Output:
- `world_state.cpp` with System message early-return guard
- `game_script.cpp` with RAID type string and WHISPER_INFORM event fire
</objective>
