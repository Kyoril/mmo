<objective>
Add all missing Lua event handlers, whisper sticky-reply state management, RAID chat type support, and localization strings for the chat system UI layer.

Purpose: The client `world_state.cpp` already fires `CHAT_MSG_WHISPER`, `CHAT_MSG_RAID`, and `CHAT_MSG_SYSTEM` events, but `ChatFrame.lua` has no registered handlers for any of them. Without this plan, received whispers, raid messages, and system error messages are silently swallowed. Additionally: the `/w` slash command parser stub is commented out so whispers can never be initiated from the chat input, `ChatFrame_SendMessage` never passes the whisper target as a third argument, and no RAID ChatTypeInfo or slash commands exist.

This plan is pure Lua and data — no C++ dependency, runs in parallel with Plan 01.

Output:
- `ChatFrame.lua` with 4 new event handlers, whisper state variables, RAID TypeInfo, fixed ParseText/SendMessage/OpenChat
- `SlashCommandStrings.lua` with SLASH_RAID1/SLASH_RAID2
- All 4 `Localization.txt` files with CHAT_FORMAT_WHISPER_FROM/TO, CHAT_FORMAT_RAID, CHAT_TYPE_WHISPER, CHAT_TYPE_RAID
</objective>

function ChatFrame_SendMessage(this)
