## Partial Requirements — Key Issues

### GRP-03 / LOOT-04 — Chest Group Loot
**File:** `src/shared/game_server/objects/game_world_object_s.cpp:88–95`  
Chest `LootInstance` is constructed with one recipient and no loot method. Group loot rules (RoundRobin, MasterLoot) only apply to creature corpses.  
**Fix:** Before creating LootInstance, gather nearby group members the same way `creature_ai_death_state.cpp` does; pass loot method + master GUID.

### EDIT-02 / WOBJ-03 — Per-Spawn trigger_id Runtime Wiring
**File:** `src/shared/game_server/world/world_instance.cpp:141`  
`spawn.trigger_id()` is not passed to `WorldObjectSpawner`. The proto field and editor UI exist but the value is silently dropped.  
**Fix:** Extend `WorldObjectSpawner` ctor with `uint32 triggerIdOverride = 0`; call `spawned->SetTriggerIdOverride()` in `SpawnOne()`; fire trigger in `game_world_object_s.cpp::Use()` when `m_triggerIdOverride != 0`.
